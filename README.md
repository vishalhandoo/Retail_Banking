# Retail Banking Leave-Risk Analysis

This project analyzes a retail banking customer dataset to understand service-quality drivers of customer retention and identify customers who are most likely to leave. It combines SERVQUAL-based service metrics, business thresholds, machine learning outputs, customer-level next-best-action logic, and a lightweight dashboard with Decipher AI for grounded business Q&A.

## Project Overview

The workflow in this repository:

- loads the `Retail_Banking_Dataset_JKBank.csv` dataset
- builds SERVQUAL-based service quality measures
- creates a leave-risk proxy target from low retention scores because the source `churn_flag` does not contain observed leavers
- trains a model to estimate customer leave propensity
- exports summary tables and high-risk customer lists
- serves a browser-based dashboard for portfolio review and customer-level follow-up
- includes a Customer 360 Review modal for high-priority customers
- includes Decipher AI, a grounded assistant that explains KPIs, charts, leave-risk patterns, and retention actions using the project's summary outputs

## Key Highlights

- Model accuracy: `85.65%`
- Model precision: `67.63%`
- Model recall: `81.70%`
- Strongest SERVQUAL predictor: `servqual_responsiveness`
- Customers with `resolution_days > 6.5` show about `6.28x` higher leave risk than the rest
- Customers with a relationship manager have lower predicted leave probability (`22.64%`) than those without one (`34.63%`)

## Dashboard Features

The local dashboard includes these main pages:

- `Overview`: portfolio-level snapshot of customers, risk, service quality, and retention workload
- `Health Check`: plain-English KPI cards and definitions
- `Key Issues`: risk drivers such as complaint delay, low app rating, and relationship-manager gaps
- `Charts`: visual diagnostics generated by the analysis pipeline
- `Action List`: highest-priority customers sorted by risk-adjusted retention score
- `Decipher AI`: grounded dashboard Q&A using the exported files and current filter context
- `Details`: supporting analyst tables
- `Files`: downloadable generated CSV outputs

## Customer 360 Review

The Action List supports customer-level review. Click a `Customer ID` in the high-priority table to open the Customer 360 Review modal.

The modal explains:

- why the customer is in the current risk category
- predicted leave probability, retention score, service quality index, app rating, complaint count, resolution days, credit score, product count, RM status, and credit-review status
- warning signals such as slow complaint resolution, weak app rating, no RM assigned, low service quality, or weak credit score
- RM next steps, including same-day contact for `Critical` customers
- recommended support such as service recovery callback, EMI date alignment discussion if policy allows, digital support, auto-pay setup, and eligible rate or tenure review
- offers to avoid, especially aggressive unsecured lending or generic cross-sell before service recovery or credit review

The Customer 360 logic is deterministic and local. If a field is unavailable in `final_project_output.csv`, the dashboard shows `NA` instead of inventing a value.

## Decipher AI

The dashboard includes Decipher AI, an internal assistant for the dashboard handler. It is designed to:

- answer questions about why customers are likely to leave
- explain charts and KPIs in plain business language
- summarize risk patterns from the current dashboard view
- suggest retention actions such as faster complaint resolution, app-experience improvements, stronger responsiveness and reliability, targeted outreach to high-risk customers, and selective relationship-manager expansion

Decipher AI is grounded in:

- the exported summary CSV files
- the filtered customer view currently open in the dashboard
- the report draft `SIP_Report_Full_Draft.md`

It does not require a separate SDK package because the Gemini integration uses Python's standard library HTTP client. The Decipher AI page is also protected from dashboard auto-refresh while you are chatting, and chat messages are kept in browser session storage for the current tab.

## How To Run

1. Install dependencies:

```bash
pip install pandas numpy matplotlib seaborn scikit-learn
```

2. Optional but recommended for Decipher AI: add your Gemini settings to the local `.env` file in the project root.

Example `.env`:

```env
GEMINI_API_KEY=your_api_key_here
GEMINI_MODEL=gemini-2.5-flash
```

The dashboard loads `.env` automatically on startup, so this is the easiest setup path.

You can still use normal OS environment variables if you prefer.

PowerShell:

```powershell
$env:GEMINI_API_KEY="your_api_key_here"
```

Command Prompt:

```cmd
set GEMINI_API_KEY=your_api_key_here
```

Optional model override:

```powershell
$env:GEMINI_MODEL="gemini-2.5-flash"
```

If `GEMINI_API_KEY` is not set in either `.env` or your OS environment, the dashboard still runs. The Decipher AI section shows a helpful configuration message instead of crashing, and the deterministic dashboard and Customer 360 features continue to work.

3. Put the dataset in one of these locations:

- `../Retail_Banking_PrivateData/` from the repo folder
- the folder pointed to by the `RETAIL_BANKING_DATA_DIR` environment variable
- the project folder as a fallback

4. Run the analysis:

Make sure one of these dataset files is available:

- `Retail_Banking_Dataset_JKBank.csv`
- `Enhanced_Retail_Banking_Dataset.csv`

```bash
python retail_banking_analysis.py
```

5. Start the dashboard:

```bash
python retail_banking_dashboard.py
```

6. Open the dashboard in your browser:

```text
http://127.0.0.1:8501
```

7. Open the `Action List` page and click a `Customer ID` to test the Customer 360 Review modal.

8. Open the `Decipher AI` page in the dashboard navigation and test one of the example prompts.

Suggested demo questions:

- `Why are customers likely to leave?`
- `Which factors most affect retention?`
- `What does the complaint-resolution chart mean?`
- `Which customers need urgent intervention right now?`
- `How can the bank improve retention rate based on this dashboard?`

## GitHub Pages

GitHub Pages serves static files, so export the dashboard before publishing:

```bash
python retail_banking_dashboard.py --export-static
```

This writes the site to `docs/`, including:

- `index.html`
- one HTML file for each dashboard page
- chart images under `docs/assets/charts/`
- public-safe summary CSV downloads under `docs/assets/downloads/`

Customer-level exports such as `final_project_output.csv` and `likely_to_leave_customers.csv` are intentionally kept out of the static GitHub Pages export.

Privacy note: the rendered dashboard pages can still include visible customer-level rows or Customer 360 details from the latest local output. Review the generated `docs/` site before publishing it publicly if the underlying customer data is sensitive.

Note: Decipher AI is disabled in the static export because GitHub Pages cannot host the live Gemini API endpoint used by the local dashboard server.

Commit the generated `docs/` folder and push to `main`. The included GitHub Actions workflow publishes `docs/` to GitHub Pages automatically.
